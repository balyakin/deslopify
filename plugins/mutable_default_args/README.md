# Плагин: Проверка изменяемых значений по умолчанию (Mutable Default Args)

Этот плагин находит классический антипаттерн Python — использование изменяемых (mutable) структур данных (таких как `list`, `dict` или `set`) в качестве значений по умолчанию для аргументов функции.

## Почему это плохо?
В Python значения по умолчанию (default arguments) вычисляются **только один раз** — в момент определения (парсинга) функции, а не при каждом ее вызове.
Если в качестве значения по умолчанию используется список `[]`, и функция модифицирует этот список, то при следующем вызове функции (без передачи этого аргумента) в списке **уже будут лежать данные от предыдущего вызова**. 

Это приводит к крайне непредсказуемым "плавающим" багам, когда данные протекают между совершенно не связанными вызовами функций. Языковые модели (LLM) часто допускают эту ошибку при написании кода "на автомате".

## Что делает плагин
- Находит все определения обычных (`def`) и асинхронных (`async def`) функций.
- Проверяет все значения по умолчанию (как позиционные, так и keyword-only аргументы).
- Если значением по умолчанию является литерал `[]` (список), `{}` (словарь), литерал множества, либо прямой вызов `list()`, `dict()`, `set()`, плагин генерирует ошибку.

**Правильный паттерн**, который следует использовать вместо этого:
```python
def process_data(items=None):
    if items is None:
        items = []
    # ...
```
