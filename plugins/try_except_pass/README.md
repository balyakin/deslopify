# Плагин: Проверка на пустое подавление исключений (try/except pass)

Этот плагин находит блоки `try/except`, в которых обработчик исключения (`except`) состоит исключительно из оператора `pass`.

Подобный паттерн ("поглощение ошибки в ноль") часто генерируется языковыми моделями (LLM) как временная заглушка, но в реальном коде (production) он приводит к скрытому подавлению ошибок, из-за чего диагностика проблем становится невозможной.

## Что делает плагин
- Анализирует все обработчики исключений (узлы `ExceptHandler` в AST).
- Проверяет, состоит ли тело обработчика ровно из одной инструкции.
- Если эта инструкция — `pass`, плагин генерирует ошибку.

Файлы тестов (начинающиеся с `test_`) автоматически игнорируются при проверке, так как в тестах иногда намеренно подавляют исключения.
