# Реализованные плагины

В рамках проекта deslopify на данный момент реализованы следующие плагины (всего 27):

1. **empty_function** — находит функции, тело которых состоит только из `pass`, `...` или докстринга (кроме абстрактных методов).
2. **mutable_default_args** — выявляет использование изменяемых объектов (list, dict, set) в качестве значений аргументов по умолчанию.
3. **try_except_pass** — обнаруживает блоки `except`, которые содержат только инструкцию `pass`.
4. **unnecessary_class** — находит классы, используемые только как пространства имен (без состояния).
5. **logging_config_in_code** — запрещает настройку логирования (`basicConfig`, `dictConfig`) прямо в коде модулей.
6. **hardcoded_path** — находит жестко заданные абсолютные пути разработчика в коде.
7. **port_in_code** — запрещает использование магических чисел портов в сетевых вызовах и URL.
8. **no_uuid_in_tests** — запрещает генерацию случайных UUID в тестовых файлах.
9. **config_only_configurable** — запрещает хранение имен таблиц и колонок БД в файлах конфигурации.
10. **sqlalchemy_named_constraints** — требует обязательного наличия имен у всех констрейнтов SQLAlchemy.
11. **rpc_method_docstring** — проверяет наличие русских докстрингов у методов, начинающихся с `rpc_`.
12. **m7_service_logging_only** — требует использования `m7-service-logging` вместо стандартного `logging`.
13. **no_enum_cast** — запрещает каст Enum к `int`, предлагая использовать `.value`.
14. **datetime_isoformat** — требует использовать `.isoformat()` для преобразования дат в строки.
15. **no_method_calls_in_init** — запрещает вызовы методов `self` внутри конструктора `__init__`.
16. **sqlalchemy_datetime_timezone** — требует использовать `DateTime(timezone=True)` в моделях.
17. **string_enum_only** — требует, чтобы все Enum наследовались от `str`.
18. **no_magic_numbers** — запрещает использование магических чисел в коде (кроме констант).
19. **no_global** — полностью запрещает использование ключевого слова `global`.
20. **log_exception_in_except** — требует использования `log.exception()` в блоках `except` для сохранения трейсбека.
21. **no_returns_none_in_docstring** — запрещает избыточное `Returns: None` в докстрингах.
22. **no_string_concatenation** — запрещает склейку строк через `+`, требуя использования форматирования.
23. **async_for_execute** — рекомендует использовать `async for row in conn.execute(query)` вместо `.fetchall()`.
24. **trailing_comma_in_dict** — требует завершающую запятую в многострочных словарях.
25. **use_isoparse** — требует использования `isoparse()` из `dateutil.parser` для разбора ISO-дат.
26. **no_isinstance_bool_safe** — требует использовать `type(a) is int` вместо `isinstance(a, int)` для защиты от булевых значений.
27. **no_mutation_during_iteration** — запрещает удалять элементы из списка или словаря во время итерации по ним.
